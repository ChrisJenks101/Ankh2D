tmx2bin

This tool converts the XML-based TMX file format from the Tiled Map Editor into a binary format, this is the only file format that is dependant on tinyxml2. Binary format also future proofs any lower-end platforms that would be slowed down by parsing an XML file.

Note: This tool only converts the data that is required, this is not a full featured TMX to binary converter.

Note: In order to use collisions built in Tiled Map Editor, create a layer named "collisions" (lowercase) and use any tile to populate If there are no collisions (MAP::num collisions = 0) then you do not need to parse the final collision layer when using the map file.

MAP

Name Datatype Description
signiture char[4] The signiture for this file format "TMX."
map width uint16 The map width in tiles
map height uint16 The map height in tiles
tile width uint8 The width of a tile (px)
tile height uint8 The height of a tile (px)
num collisions uint8 The number of tilesets that have collisions on them
num tilesets uint8 The number of tilesets present in this tmx map
num layers uint8 The number of layers present in this tmx map
tilesets TILESET[num tilesets] Information about the tileset you're using to tile the map
layers LAYER[num layers] Information about the different layers the tmx map uses
collision data uint8[num collisions] Information about the collision that are present in the map (1/0)

TILESET - 40 bytes

Name Datatype Description
first Gid uint8 The first global tile ID of this tileset
name char[10] The name of the tileset in use
tile width uint16 The width of the tiles (px)
tile height uint16 The height of the tiles (px)
filename char[25] The absolute path to the texture file used in this tileset

LAYER - 16 bytes

Name Datatype Description
name char[10] The name assigned to this layer
layer width uint16 The width of the layer in tiles
layer height uint16 The height of the layer in tiles
tile id uint16 The tile index from the tileset
